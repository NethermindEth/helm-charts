A Helm chart to help define SLOs

## Usage

Check `values.yaml` for instructions on specific values.

The release name of the chart will be used as the service name and as part of alert names and SLOs. For example, given
these values:
```yaml
slos:
  availability:
    # snip
  latency:
    # snip
```

And a release name as such:
```shell
helm install some-service nethermind/slos
```
or
```yaml
helmCharts:
  - name: slos
    repo: https://nethermindeth.github.io/helm-charts
    version: "*"
    valuesFile: values.something.yaml
    releaseName: some-service
    namespace: some-namespace
```

Will yield:
- A SLO object named `some-service-slos`
- With `service` set to `some-service`
- With two SLOs/alerts:
  - `some-service-availability-SLO`
  - `some-service-latency-SLO`
